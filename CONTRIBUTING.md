# Contributing to Legiflow

Thank you for your interest in contributing to Legiflow. This project aims to make legal knowledge easier to understand and more accessible for everyone.

## Getting Started

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your changes.
4. Make your updates and test them properly.
5. Commit your changes with a clear message.
6. Push your branch and open a Pull Request.

## Branch Naming Convention

Use meaningful branch names:

* `feature/add-ipc-search`
* `fix/login-page-bug`
* `docs/update-contributing-guide`

## Commit Message Format

Write short and clear commit messages:

* feat: add agreement risk indicator`
* fix: resolve upload page alignment issue`
* docs: update README with setup steps`

## Coding Guidelines

* Use TypeScript for development.
* Keep the code clean, readable, and reusable.
* Follow consistent naming conventions.
* Add comments only when necessary.
*  Avoid unnecessary dependencies.
* Ensure the UI remains responsive and accessible.

## Pull Request Guidelines

Before submitting a Pull Request:

* Test your changes locally.
* Ensure there are no build or lint errors.
* Keep the PR focused on a single issue.
* Add screenshots for UI-related changes.
* Clearly describe what changes were made.

## Documentation Guidelines

If your contribution changes functionality, update the relevant documentation as well.

Examples:

* Update README if setup steps change
* Add screenshots for new UI sections
* Document new APIs, routes, or components

## Code of Conduct

Be respectful and collaborative. Constructive feedback and positive communication are encouraged for all contributors.
